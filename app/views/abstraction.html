Thinking about data

We're trying to think a bit differently about the data we currently hold (and might hold) and trying to classify it to help us understand how different data may, or may not, affect what users deem 'relevant'.

There is almost certainly lots of prior art here, we should find it

Previous research:

  • Semarchy have some thoughts but no obvious references at What is master data
  • OCDQ refers to it as master data too, but uses transaction data to cover events. Maybe they're the same. More at The circle of quality

Data might be....

Things

Data describing things, often physical things, but sometimes abstract.

  • Has an identity, often has a name or code identifying it
  • Often has a location, although this might be transient

Examples: Naptan, Libraries, CCTV.

Events

Data describing events, things that will happen or have happened.

  • Has an identity, often has a name or code identifying it
  • Has a location, everything happens somewhere
  • Has a time, this might be a date, or date range when the event will happen

Examples: Bin collection times, Planned Roadworks,

Transactions

An interaction between two or more parties, often financial.

  • Has at least 2 actors
  • Normally has a value attached
  • Has a time when the transaction took place
  • May relate to a thing

Examples: House prices, VOA valuations, Spending (various amounts), GPC.

Facts

This is probably at the wrong level of abstraction, but we're covering everything else under this until we work out what the others are.
